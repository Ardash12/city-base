Задание.
Нужно разработать веб-сервер на Django для хранения списка городов и проверки уже известных городов по названиям.
У сервера должен быть один endpoint. Он принимает в качестве запроса название города.
Если такой город есть в БД, то вернуть json со значением true.
Если город отсутствует в БД, то записать в БД, но вернуть значение false.

Для скорости работы нужно организовать кэш. Некторые города могут проверяться очень часто, и чтобы не создавать
лишнюю нагрузку на БД, можно поддерживать N значений в памяти и сравнивать с ними.
Пусть это будет простейший кэш, который хранит список из N=100 последних запрошенных городов. Как только приходит
запрос с городом, который отсутствует в кэше, нужно:
1. проверить город в БД и узнать ответ,
2. при надобности, удалить из кэша самый старый запрошенный город,
3. добавить в кэш только что запрошенный город.
Кэш должен работать одинаково быстро как для N=100, так и для N=100000000

Сделать файл с тестом, в котором будет 100000 строк с городами (многие повторяются), и скрипт, который читает
этот файл, последовательно запрашивает у сервера ответы и сохраняет ответы в выходной файл. В отдельном файле
заранее записать для каждого города, какой ответ ожидается. Чтобы можно было сравнить результат работы веб
сервера и эталонный ожидаемый результат.
